Вернемся к настоящему

Есть код

        
          let length = 12;

          if (length > 10) {
            console.log('Wow!');
            console.log('So large!');
          }

          if (length > 11) {
            console.log('Wow!');
            console.log('So large!');
          }

Есть проблема

Теперь заказчик хочет так:

        
          let length = 12;

          if (length > 10) {
            alert('Wow!');
            alert('So large!');
          }

          if (length > 11) {
            alert('Wow!');
            alert('So large!');
          }

Представьте, для того чтобы задернуть шторы, нужно просто перевесить каждый крючок каждой шторы.

Решение

Нужно вынести куда-то этот код.

        
          console.log('Wow!');
          console.log('So large!');

Тогда можно заменить его в одном месте и не грустить.

Функции

В JavaScript есть функции. Это вынесенные кусочки кода, которые можно использовать повторно.

Функцию сначала объявляют. Назвать функцию можно как угодно:

        
          function functionName() {
            // code
          }

А затем вызывают по имени:

        
          functionName();

При вызове происходит выполнение тела функции — блока кода внутри фигурных скобок.

Пример функции

        
          let length = 12;

          function sayLarge() {
            console.log('Wow!');
            console.log('So large!');
          }

          if (length > 10) {
            sayLarge();
          }

          if (length > 11) {
            sayLarge();
          }

Функция

Теперь, чтобы заменить console.log на alert достаточно сделать это только внутри функции.

        
          let length = 12;

          function sayLarge() {
            alert('Wow!');
            alert('So large!');
          }
          
          if (length > 10) {
            sayLarge();
          }
          
          if (length > 11) {
            sayLarge();
          }

Легко и удобно!

        
          function sayMyName() {
            console.log('Heisenberg');
          }
          
          sayMyName();
          sayMyName();
          sayMyName();

Выведет в консоль Heisenberg три раза.

Область видимости

У переменной есть область видимости. Это тот код, который видит переменную и может получить к ней доступ.

Определяется блоком кода вокруг объявления переменной переменной. Блоки кода обозначаются фигурными скобками.

Что будет в консоли?

        
          function sayMyName() {
            let name = 'Heisenberg';
            console.log(name);
          }
          
          let age = 53;
          if (age > 50) {
            let message = 'Okay';
            console.log(message);
            
            sayMyName();
          }
          
          console.log(name);
          console.log(message);

Вывод, пустая строка и ошибка

        
          Okay
          Heisenberg
          
          Uncaught ReferenceError: message is not defined
            at :15:13

Область видимости

Переменная доступна внутри тех фигурных скобок, где она объявлена.

Можно ее изменять из «дочерних» блоков кода?

Можно

        
          let age = 25;

          function birthday() {
            age = age + 1;
            console.log(age);
          }
          
          birthday();
          birthday();
          birthday();

Результат

        
          26
          27
          28

А что если перезаписать ее?

        
          let age = 25;

          function birthday() {
            let age = 23;
            console.log(age);
          }
          
          birthday();
          birthday();
          birthday();

Перезапишется

        
          23
          23
          23

Лучше так не делать. Это снижает читаемость кода!

И так!

Функции — именованные блоки кода.

Область видимости — код который имеет доступ до переменной.
